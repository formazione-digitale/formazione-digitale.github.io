# Mappa DigComp / DigCompEdu — Metadocumento metodologico
**Portale:** Formazione Digitale — cristianodepasquale.github.io/formazione-digitale/  
**Autore portale:** Cristiano De Pasquale  
**Scopo documento:** Definire il metodo con cui verrà prodotta la mappa delle competenze DigComp e DigCompEdu toccate dalle risorse del portale, in vista dell'integrazione in una relazione per il portfolio professionale INDIRE.  
**Stato:** Metadocumento — da eseguire quando il portale sarà sufficientemente sviluppato.

---

## 1. Obiettivo del task

Produrre un documento che mappi ogni risorsa del portale (guide, strumenti, pillole) sulle competenze dei framework europei **DigComp** (competenze digitali del cittadino) e **DigCompEdu** (competenze digitali del docente), con argomentazione esplicita e collegamento agli **8 Standard minimi INDIRE**.

Il documento prodotto sarà la **sezione descrittivo-analitica** della relazione INDIRE — la parte che collega artefatti didattici a standard di competenza professionale. La riflessione in prima persona (richiesta dal Comitato di Valutazione) rimane a cura del docente.

## 3. Metodo di esplorazione

Quando il portale sarà pronto, l'analisi seguirà questi passi nell'ordine indicato.

### Passo 1 — Inventario risorse
Navigare il portale live e costruire un inventario completo:

```
Per ogni risorsa:
- URL
- Tipo (guida / strumento / pillola)
- Titolo
- Disciplina/argomento
- Livello dichiarato
- Caratteristiche tecniche (interattività, grafici, download, template…)
```

### Passo 2 — Analisi per risorsa
Per ogni risorsa, rispondere a queste domande analitiche:

1. **Cosa fa l'utente** con questa risorsa? (legge, interagisce, scarica, simula…)
2. **Quale competenza digitale del cittadino** viene sviluppata o esercitata? → DigComp
3. **Quale competenza del docente** viene dimostrata nella progettazione/uso? → DigCompEdu
4. **Quale standard INDIRE** è attivabile tramite questa risorsa?
5. **Livello DigCompEdu** raggiungibile (A1–C2: Novizio → Pioniere)?

### Passo 3 — Matrice di copertura
Costruire una matrice che mostri a colpo d'occhio:

- Righe: risorse del portale
- Colonne: aree DigCompEdu (1–6) + aree DigComp (1–5)
- Celle: ✅ coperta con argomentazione / ⬜ non coperta / 🔶 parzialmente

### Passo 4 — Gap analysis
Identificare le aree DigCompEdu non ancora coperte dal portale, con indicazione di quale tipo di risorsa futura le coprirebbe.

### Passo 5 — Testo per relazione INDIRE
Produrre per ogni standard INDIRE attivato un paragrafo in forma discorsiva, direttamente integrabile nella relazione, con questa struttura:

```
[Standard INDIRE Ax] — [Titolo standard]
La risorsa "[titolo]" dimostra il possesso di questo standard attraverso [descrizione
della scelta progettuale]. In termini DigCompEdu, l'artefatto attiva la competenza
[X.Y — nome] in quanto [argomentazione]. Sul versante DigComp, gli studenti
esercitano la competenza [area.n] attraverso [descrizione dell'attività].
```

## 4. Output atteso

Il documento finale conterrà:

1. **Inventario annotato** di tutte le risorse del portale
2. **Scheda per risorsa** con mapping DigComp + DigCompEdu + Standard INDIRE
3. **Matrice di copertura** visuale (tabella)
4. **Gap analysis** — aree non ancora coperte
5. **Paragrafi pronti** per la relazione INDIRE, uno per standard attivato

## 5. Note operative

- L'analisi si basa sul portale live al momento dell'esecuzione — non sullo stato attuale.
- Ogni argomentazione fa riferimento al testo originale del framework DigCompEdu (traduzione CNR-ITD, 2018).
- Il livello DigCompEdu stimato è indicativo: la certificazione formale richiede autovalutazione tramite strumento ufficiale (SELFIE for Teachers o equivalente MIM).
- Il documento prodotto è destinato all'uso personale nel portfolio professionale — non costituisce certificazione.
